# KKI2026 Monorepo Design

**Tanggal:** 2026-07-20  
**Status:** Disetujui untuk implementasi lokal; remote GitHub dibuat dan dipush oleh pengguna.

## Tujuan

Memindahkan bagian software proyek ASV KKI 2026 ke `D:/KKI2/KKI2026` sebagai satu repository lokal yang dapat diuji dari satu root, tanpa memindahkan atau mengubah path runtime yang sudah dipakai oleh dashboard dan bridge Raspberry Pi.

## Batasan repository

Yang masuk repository:

- `dashboard/` TanStack Start frontend;
- `asv_dashboard_backend/` FastAPI bridge Raspberry Pi;
- `vision_test.py` dan `vision_route.py`;
- `tests/`;
- `supabase/migrations/`;
- `deploy/raspberry-pi/`;
- `requirements-dashboard*.txt`;
- dokumentasi software di `docs/`;
- konfigurasi workspace dan ignore rules.

Yang tidak masuk repository software-only:

- CAD FreeCAD/DXF/FBX/OBJ/MTL dan backup CAD;
- PDF panduan lomba;
- model YOLO/NCNN, `model.zip`, dan artefak model;
- gambar hasil eksperimen dan log `*.jsonl`;
- cache/build output, virtual environment, secret, dan `.env.local`.

## Arsitektur

Root memakai NPM workspaces dengan satu workspace JavaScript: `dashboard/`. Backend Python tetap berada di root sebagai package/runtime monorepo terpisah; tidak dipaksa menjadi package NPM dan tidak dipindahkan ke `apps/`.

```text
KKI2026/
├── dashboard/                 # NPM workspace: TanStack Start
├── asv_dashboard_backend/    # Python FastAPI bridge
├── tests/                    # Python tests
├── supabase/migrations/      # Database/realtime schema
├── deploy/raspberry-pi/      # systemd, env, tunnel templates
├── docs/                     # Software specifications and plans
├── vision_route.py           # Pure computer-vision route primitives
├── vision_test.py            # Existing model/Pixhawk entry point
├── package.json              # Root workspace and orchestration scripts
├── package-lock.json         # Single NPM lockfile
├── requirements-dashboard.txt
└── .gitignore
```

Root scripts delegate to the existing dashboard scripts and Python commands:

- `npm run dev` — dashboard development server;
- `npm run build` — local dashboard build;
- `npm run test` — frontend tests with the stable single-worker settings;
- `npm run lint` and `npm run typecheck` — frontend checks;
- `npm run backend:test` — Python regression tests;
- `npm run backend:compile` — Python syntax check;
- `npm run check` — frontend and backend verification sequence;
- `npm run backend:dev` — local FastAPI bridge.

No Turborepo is added: the repository has one JavaScript package and one Python service, so root NPM scripts are sufficient and easier to operate on the Raspberry Pi.

## Git handoff

The target directory receives a local Git repository, an initial commit, and the user-provided `origin` remote `https://github.com/Darelrk/KKI2026.git`. No `git push` is performed; the repository remains ready for the user to push when desired.

## Acceptance criteria

- `D:/KKI2/KKI2026` is the software-only monorepo root;
- root NPM workspace resolves `dashboard`;
- dashboard package keeps its existing scripts and imports;
- Python bridge starts and tests from the new root;
- local and Vercel frontend builds succeed from the workspace;
- generated outputs and secrets are ignored;
- local Git history exists, the user-provided `origin` is configured, and no push has been performed;
- source directory `D:/KKI2` remains available for non-software assets and is not recursively copied into the target.
